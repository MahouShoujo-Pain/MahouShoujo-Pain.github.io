Java学习笔记

数据通信三要素
  • 协议 （数据传输）
  • IP （定位主机）
  • 端口 （定位应用）
    • 逻辑地址
    • 2个字节（0~65565）
    • 1024之前的端口号软件不能自己指定（已经分配）
    • 常用
      • 80 HTTP
      • 3306 MySQL
      • 443 HTTPS
C/S 架构
TCP
  • 面向连接，需要建立连接
    • 服务器先启动，等待客户端建立会话
    • 客户端请求：3次握手，建立会话
    • 连接中包含一个对象：I/O对象（字节流）
    • 通过I/O对象进行通信
Socket & Server Socket
  1. 创建一个Socket对象绑定IP和端口
  2. 通过getOutputStream()获取网络输出流
  3. 使用网络输出流.write()发送数据
  4. 通过getInputStream()获取网络输入流
  5. 通过网络输入流,read方法获取数据
  6. 关闭Socket
  • 必须使用 网络IO流
  1. 创建一个服务器ServerSocket对象
  2. 使用accept方法获取请求连接的客户端Socket
  3. 使用获取的Socket对象获取输入流对象并使用。read（）读取数据
  4. 使用获取的Socket对象获取输出流对象并使用。write（）读取数据
  5. 释放资源.close()
文件上传案例
  1. 客户端使用本地字节输入流读取本地文件
  2. 客户端使用网络字节输出流上传数据到服务器
  3. 服务器使用网络字节输入流读取数据
  4. 服务器使用本地字节输出流保存数据
  5. 服务器使用网络字节输出流返回结果
  6. 客户端使用网络字节输入流读取返回结果
  7. 释放资源
